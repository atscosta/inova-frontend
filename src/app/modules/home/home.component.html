Nossa solução

Nosso projeto se baseia em uma coleção gerenciável de validadores que é executada para cada processo remetido ao Datajud. Classificamos nossos validadores em seis tipos: Assuntos, Classes, Movimentos, Unidades Judiciárias, Campos Obrigatórios e Outros.

A navegação pela aplicação é simples e amigável. Organizamos uma estrutura hierárquica de justiça, iniciando pelo Tipo de Justiça, em seguida o Tribunal e, por último, a Unidade Judiciária. Neste último nível, ainda oferemos os filtros por Tribunal, Tipo Unidade e Classificação. Logo abaixo, são exibidas as informações de processos de cada Unidade.

Finalmente, podemos acionar o início do processamento de validações ou detalhar os processos. As validações acontecem através de processamento paralelo e assíncrono, assim, exibimos em tempo real o progresso e a taxa de sucesso. Toda essa informação, consolidada de milhares de processos, é exibida de forma muito eficiente em gráficos que permitem a fácil leitura e organização de ações e mutirões para saneamento dos dados.

# Funcionalidades

Esta primeira versão da aplicação conta com 20 validadores já implementados e permite o fácil gerenciamento de "liga/desliga". No entanto, já identificamos e estamos planejando outras funcionalidades que podem ser facilmente adicionadas.

## Disponíveis
  • 20 validadores implementados nas seguintes categorias: Assuntos, Classes, Movimentos, Unidades Judiciárias e Campos Obrigatórios
  • Descrição e gerenciamento de "liga/desliga" de cada validador
  • Execução das validações de processos por Unidade Judiciária ou Processo inidividual
  • Acompanhamento em tempo real do andamento do processamento
  • Detalhamento dos resultados para cada processo
## Em planejamento
  • Agendamento para execução de validação
  • Endpoint público de validação processual para acesso pelos tribunais
  • Execução e visualização das validações para todas as unidades de um tribunal

# Arquitetura

A aplicação foi construída no modelo backend + frontend. No backend, usamos Java 11 e o framework Spring Boot, habilitando o Spring Webflux para uso do paradigma reativo. As validações são registradas em tempo de execução através do processamento da anotação @Validator. Cada validação é executada de forma paralela e assíncrona, definindo uma janela de até 10 threads.

No frontend, utilizamos Angular v10, com os frameworks Prime NG, Bootstrap e RxJS para as chamadas reativas.

Além da aplicação, usamos um banco de dados Mongo DB que, além de ser orientado a documentos, isto é, NoSQL, oferece suporte à programação reativa na camada de acesso a dados.

# Preparação dos dados

Além da amostra de dados processuais fornecida pelo CNJ para este Desafio, utilizamos como fonte de dados as tabelas disponibilizadas pelo Sistema de Gestão de Tabelas Processuais Unificadas (SGT) e do Painel de Produtividade Mensal - Módulo de Produtividade Mensal do Sistema de Estatísticas do Poder Judiciário (SIESPJ).
